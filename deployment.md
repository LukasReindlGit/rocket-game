# Deployment: Docker Compose + Let’s Encrypt (nginx-proxy / acme-companion)

This guide deploys **Marketing Time Game** on your own server with **HTTPS** using the common **nginx-proxy** + **acme-companion** stack (the “Let’s Encrypt companion” images from [nginxproxy](https://github.com/nginx-proxy/nginx-proxy)).

## What you get

| Piece | Role |
|--------|------|
| **nginxproxy/nginx-proxy** | Reverse proxy; reads `VIRTUAL_HOST` / `VIRTUAL_PORT` from app containers and routes HTTP(S) to them |
| **nginxproxy/acme-companion** | Requests and renews **Let’s Encrypt** certificates; watches containers with `LETSENCRYPT_HOST` |
| **marketing-time-game** | Node/Express app on port **3000** inside the network (not published directly) |

TLS certificates live in the named volume `nginx-certs`. Leaderboard CSV is persisted on the host at `./data` (mounted into the container).

## Prerequisites

1. **Server** with Docker Engine and Docker Compose v2 (`docker compose`).
2. **DNS**: An **A** (or **AAAA**) record for your chosen hostname (e.g. `game.example.com`) pointing to the server’s public IP **before** you rely on Let’s Encrypt.
3. **Ports 80 and 443** open on the host firewall (required for HTTP-01 validation and HTTPS).

## One-time setup on the server

### 1. Copy the project

```bash
cd /opt   # or your preferred path
git clone <your-repo-url> marketing_time_game
cd marketing_time_game
```

### 2. Configure environment

```bash
cp .env.example .env
nano .env   # or your editor
```

Set:

- **`APP_HOST`** — exact hostname users will use (must match DNS).
- **`LETSENCRYPT_EMAIL`** — email for ACME account and expiry notifications.
- **`SURVEY_HMAC_SECRET`** — long random secret (required in production for survey URL signing). Example:

  ```bash
  openssl rand -hex 32
  ```

### 3. Data directory

Ensure the leaderboard mount exists (Compose will create `./data` if missing; the app creates `leaderboard.csv` on first run):

```bash
mkdir -p data
```

### 4. Build and start

```bash
docker compose up -d --build
```

First start: **nginx-proxy** and **acme-companion** pull images; the companion may take **1–2 minutes** to obtain the first certificate. Check logs:

```bash
docker compose logs -f acme-companion
docker compose logs -f marketing-time-game
```

### 5. Open the app

- **Game:** `https://<APP_HOST>/game`
- **Survey:** `https://<APP_HOST>/survey?...` (as generated by the game)

HTTP on port 80 is typically redirected to HTTPS once certificates exist.

## Already running nginx-proxy elsewhere?

If the host already has **nginx-proxy** + **acme-companion** (or you manage TLS another way), you do **not** need the proxy services in this repo’s compose file.

Run only the app, with the same environment variables the proxy expects:

| Variable | Purpose |
|----------|---------|
| `VIRTUAL_HOST` | Hostname(s), comma-separated if several |
| `VIRTUAL_PORT` | **3000** (app listens here) |
| `LETSENCRYPT_HOST` | Same as `VIRTUAL_HOST` if using acme-companion |
| `LETSENCRYPT_EMAIL` | ACME contact email |

Example **override** (strip proxy services from the default file or use a small compose that only defines `marketing-time-game`):

```yaml
services:
  marketing-time-game:
    build: .
    environment:
      - VIRTUAL_HOST=${APP_HOST}
      - LETSENCRYPT_HOST=${APP_HOST}
      - LETSENCRYPT_EMAIL=${LETSENCRYPT_EMAIL}
      - VIRTUAL_PORT=3000
      - PORT=3000
      - SURVEY_HMAC_SECRET=${SURVEY_HMAC_SECRET}
    volumes:
      - ./data:/app/data
    expose:
      - "3000"
    restart: unless-stopped
```

Attach the service to the **same Docker network** as `nginx-proxy` (e.g. `docker network connect <proxy_network> <container>` or `networks: [proxy_net]` in compose).

## Updating the application

```bash
cd /path/to/marketing_time_game
git pull
docker compose up -d --build
```

Leaderboard data in `./data/leaderboard.csv` is unchanged (bind-mounted).

## Troubleshooting

| Symptom | What to check |
|---------|----------------|
| **502 Bad Gateway** | App not listening: `docker compose logs marketing-time-game`. Ensure `PORT=3000` and `VIRTUAL_PORT=3000`. |
| **Certificate stuck / HTTP only** | DNS must resolve to this server. Port 80 must reach nginx-proxy. Logs: `docker compose logs acme-companion`. |
| **Survey links invalid** | `SURVEY_HMAC_SECRET` must stay stable; changing it invalidates old QR tokens. |
| **Rate limits (Let’s Encrypt)** | Use [staging](https://letsencrypt.org/docs/staging-environment/) while testing (acme-companion docs: `ACME_CA_URI`). |

## Security notes

- Never commit `.env` with real secrets.
- Keep **`SURVEY_HMAC_SECRET`** private and backed up with your server secrets.
- The app warns if `SURVEY_HMAC_SECRET` is unset; production should always set it explicitly.
